# API Endpoint Contracts: Social Login (OAuth2)

**Feature**: Email Integration and Social Login Planning
**Date**: 2026-03-23
**Status**: Planning (Part 2 - Not Implemented)

This document specifies the API endpoint contracts for OAuth2 social login functionality with Google and Meta (Facebook) providers.

**Note**: Email integration (Part 1) uses existing auth endpoints (`POST /api/v1/auth/signup`, `POST /api/v1/auth/forgot-password`) and requires NO new endpoints.

---

## Endpoint 1: POST /api/v1/auth/social-login

Exchanges an OAuth2 authorization code for Morii Coffee JWT tokens after validating with the OAuth2 provider.

### Request

**Method**: `POST`
**Path**: `/api/v1/auth/social-login`
**Content-Type**: `application/json`
**Authorization**: None (public endpoint)

**Request Body**:
```json
{
  "provider": "google",
  "code": "4/0AeanS0...",
  "redirectUri": "https://moriicoffee.com/auth/callback",
  "state": "random_csrf_token_123"
}
```

| Field | Type | Required | Description | Validation |
|-------|------|----------|-------------|------------|
| `provider` | `string` | Yes | OAuth2 provider name | Must be one of: `google`, `meta` |
| `code` | `string` | Yes | Authorization code from OAuth2 provider | Non-empty string; single-use |
| `redirectUri` | `string` | Yes | Redirect URI used in authorization request | Must match configured redirect URI |
| `state` | `string` | Yes | CSRF token generated by client | Validated against cached state value |

### Response (Success - 200 OK)

**Content-Type**: `application/json`

```json
{
  "accessToken": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "refreshToken": "opaque_refresh_token_string",
  "user": {
    "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
    "email": "user@example.com",
    "userName": "user123",
    "fullName": "John Doe",
    "avatarUrl": null,
    "status": "Active",
    "roles": ["CUSTOMER"],
    "externalProvider": "Google",
    "hasPassword": false
  }
}
```

| Field | Type | Description |
|-------|------|-------------|
| `accessToken` | `string` | JWT access token (expires in 60 minutes) |
| `refreshToken` | `string` | Opaque refresh token (expires in 7 days) |
| `user` | `object` | User profile information |
| `user.id` | `guid` | User ID |
| `user.email` | `string` | User email address |
| `user.userName` | `string` | User username |
| `user.fullName` | `string` | User full name |
| `user.avatarUrl` | `string?` | User avatar URL (nullable) |
| `user.status` | `string` | User account status (`Active`, `Inactive`) |
| `user.roles` | `string[]` | User roles (e.g., `CUSTOMER`, `STAFF`, `ADMIN`) |
| `user.externalProvider` | `string` | Linked OAuth2 provider (`None`, `Google`, `Meta`) |
| `user.hasPassword` | `bool` | Whether user has set a password |

### Response (Error - 400 Bad Request)

**Scenario 1: Invalid Authorization Code**
```json
{
  "error": "INVALID_AUTHORIZATION_CODE",
  "message": "The authorization code is invalid or expired."
}
```

**Scenario 2: Invalid Provider**
```json
{
  "error": "INVALID_PROVIDER",
  "message": "Provider must be one of: google, meta."
}
```

**Scenario 3: State Parameter Mismatch (CSRF)**
```json
{
  "error": "INVALID_STATE",
  "message": "The state parameter is invalid or expired. This may indicate a CSRF attack."
}
```

**Scenario 4: Email Not Verified by Provider**
```json
{
  "error": "EMAIL_NOT_VERIFIED",
  "message": "Your email address is not verified with Google. Please verify your email before using social login."
}
```

### Response (Error - 409 Conflict)

**Scenario: Email Already Registered with Different Provider**
```json
{
  "error": "EMAIL_ALREADY_LINKED",
  "message": "This email is already registered with Meta. Please sign in with Meta or use a different email address."
}
```

### Response (Error - 503 Service Unavailable)

**Scenario: OAuth2 Provider Downtime**
```json
{
  "error": "PROVIDER_UNAVAILABLE",
  "message": "Google authentication service is temporarily unavailable. Please try again later or use email/password sign-in."
}
```

## Endpoint 2: GET /api/v1/auth/social-login/{provider}/authorization-url

Generates the OAuth2 authorization URL for redirecting the user to the provider's consent screen.

### Request

**Method**: `GET`
**Path**: `/api/v1/auth/social-login/{provider}/authorization-url`
**Authorization**: None (public endpoint)

**Path Parameters**:
| Parameter | Type | Required | Description | Validation |
|-----------|------|----------|-------------|------------|
| `provider` | `string` | Yes | OAuth2 provider name | Must be one of: `google`, `meta` |

**Query Parameters**:
| Parameter | Type | Required | Description | Example |
|-----------|------|----------|-------------|---------|
| `redirectUri` | `string` | Yes | Frontend callback URL | `https://moriicoffee.com/auth/callback` |
| `state` | `string` | Yes | Client-generated CSRF token | `random_csrf_token_123` |

### Response (Success - 200 OK)

**Content-Type**: `application/json`

**Example (Google)**:
```json
{
  "authorizationUrl": "https://accounts.google.com/o/oauth2/v2/auth?client_id=123456.apps.googleusercontent.com&redirect_uri=https%3A%2F%2Fmoriicoffee.com%2Fauth%2Fcallback&response_type=code&scope=openid+email+profile&state=random_csrf_token_123&access_type=offline&prompt=consent"
}
```

**Example (Meta)**:
```json
{
  "authorizationUrl": "https://www.facebook.com/v18.0/dialog/oauth?client_id=1234567890&redirect_uri=https%3A%2F%2Fmoriicoffee.com%2Fauth%2Fcallback&response_type=code&scope=email%2Cpublic_profile&state=random_csrf_token_123"
}
```

| Field | Type | Description |
|-------|------|-------------|
| `authorizationUrl` | `string` | Full OAuth2 authorization URL for redirect |

### Response (Error - 400 Bad Request)

**Scenario: Invalid Provider**
```json
{
  "error": "INVALID_PROVIDER",
  "message": "Provider must be one of: google, meta."
}
```

**Scenario: Invalid Redirect URI**
```json
{
  "error": "INVALID_REDIRECT_URI",
  "message": "The redirect URI must be a valid HTTPS URL."
}
```

## OAuth2 Authorization Code Flow (End-to-End)

### Flow Diagram

```
┌─────────┐                                  ┌──────────────┐                          ┌─────────────────┐
│ Browser │                                  │ Morii Coffee │                          │ OAuth2 Provider │
│ (User)  │                                  │   Backend    │                          │ (Google / Meta) │
└────┬────┘                                  └──────┬───────┘                          └────────┬────────┘
     │                                              │                                           │
     │ 1. Click "Sign in with Google"              │                                           │
     │─────────────────────────────────────────────>│                                           │
     │                                              │                                           │
     │ 2. GET /api/v1/auth/social-login/google/     │                                           │
     │    authorization-url?redirectUri=...&state=..│                                           │
     │<─────────────────────────────────────────────│                                           │
     │                                              │                                           │
     │ 3. { "authorizationUrl": "https://accounts.  │                                           │
     │    google.com/o/oauth2/v2/auth?..." }        │                                           │
     │                                              │                                           │
     │ 4. Redirect to authorization URL             │                                           │
     │───────────────────────────────────────────────────────────────────────────────────────────>│
     │                                              │                                           │
     │ 5. User signs in, grants permissions         │                                           │
     │                                              │                                           │
     │ 6. Redirect to callback URL with code        │                                           │
     │    https://moriicoffee.com/auth/callback?    │                                           │
     │    code=AUTH_CODE&state=CSRF_TOKEN           │                                           │
     │<───────────────────────────────────────────────────────────────────────────────────────────│
     │                                              │                                           │
     │ 7. POST /api/v1/auth/social-login            │                                           │
     │    { "provider": "google", "code": "...", ...}│                                          │
     │─────────────────────────────────────────────>│                                           │
     │                                              │                                           │
     │                                              │ 8. Exchange code for access token         │
     │                                              │ POST https://oauth2.googleapis.com/token  │
     │                                              │───────────────────────────────────────────>│
     │                                              │                                           │
     │                                              │ 9. { "access_token": "...", "id_token":"."}│
     │                                              │<───────────────────────────────────────────│
     │                                              │                                           │
     │                                              │ 10. Verify ID token (Google JWT)          │
     │                                              │     Extract email, sub, email_verified     │
     │                                              │                                           │
     │                                              │ 11. Check if user exists by email         │
     │                                              │     - Create or link account               │
     │                                              │     - Generate Morii Coffee JWT            │
     │                                              │                                           │
     │ 12. { "accessToken": "...", "refreshToken":  │                                           │
     │      "...", "user": { ... } }                │                                           │
     │<─────────────────────────────────────────────│                                           │
     │                                              │                                           │
     │ 13. Store tokens in Zustand authStore        │                                           │
     │     Redirect to homepage                     │                                           │
     │                                              │                                           │
```

### Detailed Steps

1. **User clicks "Sign in with Google"** on the frontend sign-in page
2. **Frontend calls** `GET /api/v1/auth/social-login/google/authorization-url`
   - Generates random `state` value (CSRF token)
   - Stores `state` in session storage
3. **Backend returns** OAuth2 authorization URL
4. **Frontend redirects** user to authorization URL (Google consent screen)
5. **User authenticates** with Google and grants permissions
6. **Google redirects** user back to frontend callback URL with `code` and `state` parameters
7. **Frontend validates** `state` parameter matches stored value (CSRF check)
8. **Frontend calls** `POST /api/v1/auth/social-login` with authorization code
9. **Backend exchanges** authorization code for access token with Google
10. **Backend validates** access token and extracts user profile (email, sub, email_verified)
11. **Backend checks** if user exists by email:
    - If exists with no external provider: link Google to existing account
    - If exists with different external provider: return 409 Conflict error
    - If not exists: create new user with Google provider
12. **Backend generates** Morii Coffee JWT access token + refresh token
13. **Backend returns** auth response with tokens and user profile
14. **Frontend stores** tokens in Zustand authStore and redirects to homepage

## Security Considerations

### State Parameter (CSRF Protection)

**Generation**:
```typescript
// Frontend (TypeScript)
const generateState = (): string => {
  const array = new Uint8Array(32);
  crypto.getRandomValues(array);
  return btoa(String.fromCharCode(...array));
};

const state = generateState();
sessionStorage.setItem('oauth2_state', state);
```

**Validation**:
```csharp
// Backend (C#)
public async Task<bool> ValidateStateAsync(string state)
{
    var cachedState = await _cache.GetStringAsync($"oauth2:state:{state}");
    if (cachedState == null)
        throw new BadRequestException("Invalid or expired state parameter");

    await _cache.RemoveAsync($"oauth2:state:{state}");  // One-time use
    return true;
}
```

### Email Verification Requirement

**Enforcement**:
```csharp
// Backend: Only link/create account if email is verified by OAuth2 provider
if (!userProfile.EmailVerified)
{
    throw new BadRequestException(
        $"Your email address is not verified with {provider}. " +
        "Please verify your email before using social login.");
}
```

**Rationale**: Prevents pre-account takeover attacks where attacker creates OAuth account with victim's email before victim registers.

### Authorization Code Exchange (Server-Side)

**Why server-side?**:
- Access token never exposed to browser during OAuth flow
- Client secret kept secure (not included in frontend JavaScript)
- Backend can enforce additional security checks (email verification, account linking rules)

**Implementation**:
```csharp
// Backend: Exchange code for token (server-to-server)
var response = await _httpClient.PostAsync(tokenUrl, new FormUrlEncodedContent(new Dictionary<string, string>
{
    ["code"] = code,
    ["client_id"] = _settings.Google.ClientId,
    ["client_secret"] = _settings.Google.ClientSecret,  // Never exposed to frontend
    ["redirect_uri"] = redirectUri,
    ["grant_type"] = "authorization_code"
}));
```

## Example Integration (Frontend)

### React Component: SocialLoginButtons

```typescript
"use client";

import { useAuthStore } from "@/stores/authStore";
import { Button } from "@/components/ui/button";

export function SocialLoginButtons() {
  const { socialLogin } = useAuthStore();

  const handleGoogleLogin = async () => {
    try {
      await socialLogin("google");
    } catch (error) {
      console.error("Google login failed:", error);
    }
  };

  const handleMetaLogin = async () => {
    try {
      await socialLogin("meta");
    } catch (error) {
      console.error("Meta login failed:", error);

  return (

        variant="outline"

  );
}
```

### Zustand Store: authStore

```typescript
interface AuthStore {
  // Existing state...
  socialLogin: (provider: string) => Promise<void>;
  handleOAuth2Callback: (code: string, state: string) => Promise<void>;
}

export const useAuthStore = create<AuthStore>((set) => ({
  // ...

  socialLogin: async (provider) => {
    // Generate CSRF token
    const state = generateRandomState();
    sessionStorage.setItem("oauth2_state", state);

    // Get authorization URL from backend
    const response = await fetch(
      `/api/v1/auth/social-login/${provider}/authorization-url?` +
      `redirectUri=${encodeURIComponent(window.location.origin + "/auth/callback")}&` +
      `state=${encodeURIComponent(state)}`
    );

    const { authorizationUrl } = await response.json();

    // Redirect to OAuth2 provider
    window.location.href = authorizationUrl;
  },

  handleOAuth2Callback: async (code, state) => {
    // Validate state parameter (CSRF check)
    const storedState = sessionStorage.getItem("oauth2_state");
    if (storedState !== state) {
      throw new Error("Invalid state parameter. Possible CSRF attack.");
    }
    sessionStorage.removeItem("oauth2_state");

    // Extract provider from URL path
    const provider = new URLSearchParams(window.location.search).get("provider");

    // Exchange code for tokens
    const response = await fetch("/api/v1/auth/social-login", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({
        provider,
        code,
        redirectUri: window.location.origin + "/auth/callback",
        state
      })
    });

    const { accessToken, refreshToken, user } = await response.json();

    // Store tokens
    set({ accessToken, refreshToken, user, isAuthenticated: true });

    // Redirect to homepage
    window.location.href = "/";
  }
}));
```

## Error Handling Matrix

| Error Scenario | HTTP Status | Error Code | Frontend Action |
|----------------|-------------|------------|-----------------|
| Invalid authorization code | 400 | `INVALID_AUTHORIZATION_CODE` | Show error message; offer retry |
| Invalid provider | 400 | `INVALID_PROVIDER` | Show error message; log bug |
| State parameter mismatch | 400 | `INVALID_STATE` | Show CSRF warning; log security event |
| Email not verified | 400 | `EMAIL_NOT_VERIFIED` | Prompt user to verify email with provider |
| Email already linked (different provider) | 409 | `EMAIL_ALREADY_LINKED` | Suggest using original provider or contacting support |
| OAuth2 provider downtime | 503 | `PROVIDER_UNAVAILABLE` | Show temporary error; suggest email/password login |
| Network timeout | 408 | `REQUEST_TIMEOUT` | Retry with exponential backoff; fallback to email/password |
